T-shirt sizing: **XS** (trivial) · **S** (small, focused) · **M** (moderate, multiple moving parts) · **L** (large, cross-cutting or high risk)

Developers are expected to create sub-tasks for implementation detail.

### Task 5 — Shared query utilities

Complete this task before implementing any repository method. All four utilities are prerequisites.

|             Sub-task              | Size |                                                                                                                                                                                                                                 Notes                                                                                                                                                                                                                                 |
|-----------------------------------|------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| `AgentBaselineFilterBuilder`      | S    | Static `build(AgentQueryParams)` method. Always includes: `state = COMPLETED`, `startDate` range, `tenantId IN [...]`, nested `exists` on `agentInstances.agentInstanceKey`. Conditionally adds `processDefinitionKey` term when non-null. Both ES and OS implementations call this builder — write it against a shared query DSL abstraction or duplicate per engine if no shared layer exists. Tenant filter is a security requirement; missing it is a regression. |
| `PeriodComparisonExecutor`        | S    | Runs current and prior period queries in parallel via `CompletableFuture`. Returns `PeriodComparisonResult<T>` record. Prior period = same duration shifted back (`forPreviousPeriodWindow()`). Used only by `getSummary`. Wire with a dedicated `Executor` bean to avoid blocking the request thread.                                                                                                                                                                |
| `DateIntervalResolver`            | XS   | Static `resolve(Instant from, Instant to)` returns `"1h"` / `"1d"` / `"1w"` / `"1M"` based on range in days (≤2 / ≤30 / ≤180 / else). Used by `/trends` to set `calendar_interval`.                                                                                                                                                                                                                                                                                   |
| `IncidentRateHelper`              | XS   | Static `computeRate(long incidentCount, long totalRuns)` → `totalRuns > 0 ? (double) incidentCount / totalRuns : 0.0`. Shared by `getSummary` and `getCharts` (incident rate by version).                                                                                                                                                                                                                                                                             |
| Unit tests for all four utilities | S    | Cover: `AgentBaselineFilterBuilder` with and without `processDefinitionKey`; `PeriodComparisonExecutor` parallel execution and result pairing; all four `DateIntervalResolver` branches; `IncidentRateHelper` zero-denominator guard.                                                                                                                                                                                                                                 |

### Task 6 — `GET /acp/summary` endpoint

|                    Sub-task                    | Size |                                                                                                                                                                        Notes                                                                                                                                                                         |
|------------------------------------------------|------|------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| `AgenticControlPlaneRepositoryES.getSummary()` | M    | ES query with `value_count` (totalRuns), `avg` (duration), `percentiles` at [50, 95] (duration), `sum` (agentTotalInputTokens, agentTotalOutputTokens), `percentiles` via script for medianTokens, nested `filter` for has_incidents count. Map raw aggregation response to a `SummaryResult` intermediate. See spec §5.1 for full aggregation JSON. |
| `AgenticControlPlaneRepositoryOS.getSummary()` | M    | OpenSearch equivalent. Date fields in Painless return `ZonedDateTime` — use `.toInstant().toEpochMilli()` not `.getMillis()` in the `medianTokens` percentile script.                                                                                                                                                                                |
| `AgenticControlPlaneServiceImpl.getSummary()`  | S    | Call `PeriodComparisonExecutor.execute(params, repo::getSummary)`. Compute `*Delta` fields as `current.<field> - prior.<field>`. Map `SummaryResult` → `SummaryResponse`. `avgTokensPerRun` = `(totalInput + totalOutput) / totalRuns` (guard divide-by-zero → 0). `activationCount` = `totalRuns` at L0/L1.                                         |
| Wire `getSummary` in controller                | XS   | Parse `startDateFrom` / `startDateTo` as ISO-8601 `Instant`. Resolve tenant list from JWT. Construct `AgentQueryParams`. Delegate to service.                                                                                                                                                                                                        |
| Integration test                               | M    | Cover: L0 (no process key), L1 (with process key), empty date range (all zeros), delta sign (current > prior and current < prior).                                                                                                                                                                                                                   |
