# MenuButton — Carbon → shadcn migration guide

## Naming pitfall (recap)

Carbon has two "menu" components, mapped differently in shadcn:
- **Carbon `Dropdown`** = single-select listbox → **shadcn `Select`** (covered in `dropdown/`).
- **Carbon `MenuButton`** + **Carbon `OverflowMenu`** = action menu (clickable items that fire callbacks) → **shadcn `DropdownMenu`**. **This guide.**

Carbon `MenuButton` and Carbon `OverflowMenu` differ only in trigger appearance: `MenuButton` has a labelled button trigger ("Actions" + chevron), `OverflowMenu` has a `⋮` icon-only trigger. Both open the same kind of menu with `MenuItem` children.

## Structural differences

Carbon's `MenuButton` is **monolithic**: one component takes a `label` prop and renders the trigger button + menu, with `<MenuItem>` children for the actions.

shadcn's `DropdownMenu` is **fully decomposed**: `<DropdownMenuTrigger asChild><Button>…</Button></DropdownMenuTrigger>` for the trigger, `<DropdownMenuContent>` for the panel, `<DropdownMenuItem>` for each action, plus optional `<DropdownMenuLabel>` / `<DropdownMenuSeparator>` / `<DropdownMenuGroup>` / `<DropdownMenuSub>` for hierarchy.

| Concept | Carbon `MenuButton` | shadcn `DropdownMenu` |
|---|---|---|
| Wrapper | `<MenuButton label="…">` | `<DropdownMenu>` (root) |
| Trigger | implicit (button with `label` + chevron) | `<DropdownMenuTrigger asChild><Button>…</Button></DropdownMenuTrigger>` (you compose) |
| Menu content panel | implicit | `<DropdownMenuContent align="…">` |
| Action item | `<MenuItem label="Edit" onClick />` | `<DropdownMenuItem onSelect>Edit</DropdownMenuItem>` |
| Destructive item | `<MenuItem kind="danger" />` | `<DropdownMenuItem variant="destructive" />` |
| Disabled item | `<MenuItem disabled />` | `<DropdownMenuItem disabled />` |
| Submenu | `<MenuItem>` with `MenuItem` children | `<DropdownMenuSub>` + `<DropdownMenuSubTrigger>` + `<DropdownMenuSubContent>` |
| Selectable item (single) | `<MenuItemSelectable selected onChange />` | `<DropdownMenuRadioGroup value onValueChange>` + `<DropdownMenuRadioItem value>` |
| Selectable item (multi) | not built-in; use `<MenuItemSelectable>` array manually | `<DropdownMenuCheckboxItem checked onCheckedChange />` |
| Group label | not modeled | `<DropdownMenuLabel>` |
| Separator / divider | `<MenuItemDivider />` | `<DropdownMenuSeparator />` |
| Keyboard shortcut hint | not modeled | `<DropdownMenuShortcut>⌘K</DropdownMenuShortcut>` |
| Trigger size variants | `size="sm" / "md" / "lg"` | restyle the inner `<Button size>` |
| Trigger kind variants | `kind="primary" / "secondary" / "tertiary" / "ghost"` | inner `<Button variant>` |
| Menu placement | `direction="top" / "bottom"`, `flipped` | `<DropdownMenuContent side align>` |

## Carbon features missing in shadcn

- **`<MenuButton label>` driving both the trigger text AND `aria-label`** — shadcn requires you to put the label inside `<Button>` (visible text); `aria-label` is automatic for buttons with visible text.
- **Built-in chevron in the trigger** — Carbon adds the dropdown chevron automatically. shadcn requires you to render it explicitly: `<ChevronDown className="ml-2 h-4 w-4" />` from `lucide-react`.
- **`size="sm" / "md" / "lg"` trigger sizing** — Carbon's prop on `MenuButton`. shadcn: pass `size` to the inner `<Button>`.
- **`kind="primary"/"secondary"/"tertiary"/"ghost"` trigger styling** — Carbon's `kind`. shadcn: `variant` on the inner `<Button>` (default/secondary/outline/ghost/destructive/link).
- **`menuAlignment`** — Carbon's compound positioning (`bottom-start`, `top-end`, etc.). shadcn splits into `side` + `align` on `<DropdownMenuContent>`.
- **`<MenuItemSelectable>` as a single component for both checkbox and radio behaviour** — shadcn splits into `<DropdownMenuCheckboxItem>` and `<DropdownMenuRadioItem>` (under `<DropdownMenuRadioGroup>`).
- **Carbon menu submenus auto-aligning** — Carbon handles overflow flipping; shadcn (Radix) does too, but submenu uses `<DropdownMenuSub>` + `<DropdownMenuSubTrigger>` + `<DropdownMenuSubContent>` triplet rather than nested `<MenuItem>`.
- **`tabTip` / `isExpanded` state** — Carbon's experimental tab-tip variant. shadcn has none.

## shadcn features missing in Carbon

- **`<DropdownMenuLabel>`** — first-class header inside the menu (e.g., "My account"). Carbon has no equivalent — use prose inside an extra `<MenuItem disabled>` or omit.
- **`<DropdownMenuShortcut>`** — keyboard-shortcut hint aligned to the right of an item (e.g., `⌘K`). Carbon has no shortcut slot.
- **`<DropdownMenuGroup>`** — group items semantically without rendering a visible label. Carbon's `MenuItem` list is a flat `<ul>`.
- **`asChild` on the trigger** — `<DropdownMenuTrigger asChild>` accepts any element (Button, IconButton, custom component, router Link). Carbon's MenuButton is one element.
- **Animation primitives** — entry/exit animations driven by `data-state` attributes; tweak with Tailwind.
- **Submenu portal** — submenus render in a portal so they don't get clipped by ancestor `overflow:hidden`.
- **`<DropdownMenuRadioGroup>` / `<DropdownMenuCheckboxItem>` with full keyboard semantics** — first-class checkbox/radio in menus with proper ARIA; Carbon's selectable items work but are less idiomatic.

## Behavioural differences

- **Open trigger** — Both: click on trigger or Enter/Space when focused.
- **Close** — Carbon: click outside, Escape, item click. shadcn: same; additionally `onCloseAutoFocus`/`onEscapeKeyDown`/`onPointerDownOutside` lifecycle hooks via Radix.
- **Item action** — Carbon: `onClick` on `<MenuItem>`. shadcn: `onSelect` on `<DropdownMenuItem>` (note: `onSelect` for keyboard + click; `onClick` works too but `onSelect` is the canonical Radix API and runs before close).
- **Focus return** — Both return focus to the trigger after close.
- **Z-index / portal** — shadcn portals to `<body>`; Carbon renders inline (which can cause clipping in scroll containers).

## Migration checklist

1. Replace `<MenuButton label="Actions">` with the canonical composition:
   ```tsx
   <DropdownMenu>
     <DropdownMenuTrigger asChild>
       <Button variant="outline">
         Actions
         <ChevronDown className="ml-2 h-4 w-4" />
       </Button>
     </DropdownMenuTrigger>
     <DropdownMenuContent align="start">…</DropdownMenuContent>
   </DropdownMenu>
   ```
2. Replace each `<MenuItem label="X" onClick={…} />` with `<DropdownMenuItem onSelect={…}>X</DropdownMenuItem>`.
3. For destructive items: `<MenuItem kind="danger">` → `<DropdownMenuItem variant="destructive">`.
4. For disabled items: `disabled` prop maps directly.
5. For separators: `<MenuItemDivider />` → `<DropdownMenuSeparator />`.
6. For submenus: replace nested `<MenuItem>` children with `<DropdownMenuSub><DropdownMenuSubTrigger>Label</DropdownMenuSubTrigger><DropdownMenuSubContent>…</DropdownMenuSubContent></DropdownMenuSub>`.
7. For `MenuItemSelectable` single-select: wrap items in `<DropdownMenuRadioGroup value={…} onValueChange={…}>` and use `<DropdownMenuRadioItem value="…">`.
8. For `MenuItemSelectable` multi-select: use `<DropdownMenuCheckboxItem checked onCheckedChange>`.
9. Translate `menuAlignment="bottom-start"` → `<DropdownMenuContent side="bottom" align="start">`.
10. Drop trigger `size`/`kind` props — restyle the inner `<Button>` instead.
11. **Don't migrate `MenuButton` to shadcn `Select`** — that's the listbox/single-select primitive, not an action menu.
