# Heading

A context-aware heading element. Renders the right `<h1>` … `<h6>` automatically based on how deeply it is nested inside `<Section>` components. Always pair with `Section` — without one, `Heading` defaults to `<h1>`.

## When to use

- **Headings that travel** — the same content block can appear at different depths of the page tree without rewriting the tag.
- **Multi-author code where outline correctness keeps slipping** — let the framework do the bookkeeping.
- **Reusable cards/widgets/dialogs** — wrap them in `<Section>` once at the call site and the headings inside snap to the right level.

## When not to use

- **A single static heading at the top of the page** — write `<h1>` directly. Less indirection for trivial cases.
- **Display-only typography** — if it's not actually a section heading, use a `<p>` with type utilities.

## Composition

- The wrapper consumes the `SectionContext` exposed by `section/section.shadcn.tsx`.
- Default typography per level: `text-4xl/3xl/2xl/xl/lg/base font-semibold tracking-tight`. Override via `className`.
- All native `<h1>` props are forwarded.
