# Header

A sticky top app bar. Composes from a small set of slots: brand name (`HeaderName`), primary navigation (`HeaderNavigation` + `HeaderMenuItem`), and a right-aligned action bar (`HeaderGlobalBar` + `HeaderGlobalAction`).

## When to use

- **Application top bar** — the persistent navigation/branding strip across product pages.
- **Stand-alone tool surfaces** — Storybook canvases, dashboards, internal admin tools.

## When not to use

- **Page-level headings** — that's `Heading` inside a `Section`.
- **Modal title bars** — use `DialogHeader` from the `dialog` wrapper.

## Composition

- `<Header>` renders a sticky `<header>` 48px tall with a bottom border.
- `<HeaderName>` is the brand link. `prefix` renders muted text before the brand (e.g. "Camunda" → "Operate").
- `<HeaderNavigation>` wraps a `<nav>` containing `<HeaderMenuItem>` rows. Items can be marked `isActive` for selected state.
- `<HeaderGlobalBar>` pushes itself to the right (`ml-auto`) and holds icon-only actions (`<HeaderGlobalAction>`).
- All link-style elements support `asChild` to swap the underlying `<a>` for a routing-library `<Link>`.
