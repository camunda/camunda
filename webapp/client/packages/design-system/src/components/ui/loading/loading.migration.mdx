# Loading — Carbon → shadcn migration guide

## Where it sits among the loading family

Carbon ships two loading primitives:
- **`InlineLoading`** — small spinner with status text + state machine (`active`/`finished`/`error`). Used next to buttons and form fields. (Covered separately.)
- **`Loading`** — the bigger, page/section-level spinner. Optional dark overlay. **This guide.**

shadcn ships **one** spinner primitive (`Spinner`) and you compose sizing, overlay, and layout yourself. Same canonical source as `inline-loading`.

## Mapping

| Concept | Carbon `Loading` | shadcn |
|---|---|---|
| Default | 88×88 spinner positioned absolute-center | `<Spinner className="size-12" />` (or whatever size) |
| Small | `small` prop (16×16) | `<Spinner />` (default `size-4` = 16px) |
| Overlay | `withOverlay` (full-screen dark backdrop) | compose absolute positioned `<div className="absolute inset-0 bg-background/80 backdrop-blur-sm">` + center the Spinner |
| Active vs. inactive | `active` prop (defaults true) | conditional render: `{isLoading && <Spinner />}` |
| `description` (a11y label) | prop | `aria-label` on the parent or `<span className="sr-only">` next to the spinner; default `aria-label="Loading"` already on `<Spinner>` |

## Carbon features missing in shadcn

- **`withOverlay`** — Carbon ships a dark full-screen overlay primitive. shadcn requires manual composition:
  ```tsx
  <div className="absolute inset-0 z-50 flex items-center justify-center bg-background/80 backdrop-blur-sm">
    <Spinner className="size-12" />
  </div>
  ```
  Wrap in `fixed inset-0` for full-page; `absolute inset-0` inside a positioned ancestor for section-level.
- **`active={false}` to render an "off" state** — Carbon keeps the spinner mounted and stops the animation. shadcn: just don't render the Spinner when not loading.
- **`small` size variant** — Carbon's binary toggle. shadcn: just `<Spinner />` (16px) vs. `<Spinner className="size-12" />` (48px) etc.
- **`description`** (a11y) — Carbon's SR-only label. shadcn's `<Spinner>` already has `aria-label="Loading"`; override via `aria-label` prop or wrap with custom `<span className="sr-only">`.
- **Carbon's specific spinner animation** — Carbon uses a dual-arc CSS animation matching IBM's design language. shadcn uses Lucide's `Loader2Icon` with `animate-spin` (single full rotation). Visually different.
- **Built-in centering inside a positioned container** — Carbon's spinner is `position: absolute` and self-centers. shadcn: you center it (typically with `flex items-center justify-center` on the parent).
- **Skeleton variant (`<LoadingSkeleton>`)** — Carbon ships skeleton states. shadcn has only the generic `skeleton` primitive.

## shadcn features missing in Carbon

- **`<Spinner>` works inline anywhere** — drop it into buttons, badges, tiles. Carbon's `Loading` is full/block-level by design.
- **Tailwind size composability** — `<Spinner className="size-3" />`, `size-4`, `size-6`, `size-8`, `size-12`, `size-16`. Continuous sizing instead of two fixed sizes.
- **Recolour via `currentColor`** — `<Spinner className="text-primary">` recolours to whichever palette token. Carbon's spinner is fixed (typically blue).

## Behavioural differences

- **Default size** — Carbon: 88px (large) or 16px (small). shadcn: 16px (`size-4`); explicitly resize for full-page use.
- **Animation rhythm** — Carbon: ~750ms full rotation, custom dual-arc. shadcn: ~1000ms full rotation, single full circle.
- **Colour** — Carbon: blue interactive token. shadcn: inherits `currentColor`.
- **Z-index of overlay** — Carbon's overlay: high (above page content). shadcn: you set `z-50` (or whatever) explicitly.
- **Scroll-lock when overlay is shown** — Carbon: doesn't lock body scroll. shadcn: doesn't lock body scroll either. If you need it (full-page blocking), add `useEffect(() => { document.body.style.overflow = 'hidden'; return () => { document.body.style.overflow = '' } }, [])` to your overlay component or use a `<Dialog>` instead.

## Migration checklist

1. Decide whether you need the overlay or not.
2. Without overlay (most common — e.g., "loading content in a section"):
   - Replace `<Loading withOverlay={false} />` with `<Spinner className="size-12" />` (or a size that matches Carbon's 88px → use `size-16` or larger).
   - Wrap in `<div className="flex h-… items-center justify-center">` for centering.
3. With overlay (full-page or section-blocking):
   ```tsx
   <div className="absolute inset-0 z-50 flex items-center justify-center bg-background/80 backdrop-blur-sm">
     <Spinner className="size-12" />
   </div>
   ```
   Use `fixed` instead of `absolute` if you want page-wide; ensure the parent is `relative` for `absolute` to work.
4. For `small`: drop the prop, use `<Spinner />` at default size (16px).
5. For `active={false}`: stop rendering the Spinner — `{isLoading && <Spinner />}`.
6. For `description="Loading data"`: pass `aria-label="Loading data"` to `<Spinner>`.
7. For full-page blocking with scroll-lock: consider switching to a shadcn `<Dialog>` with a Spinner inside instead — that's the right primitive for "block the page until done".
8. For Carbon's specific dual-arc animation: not preserved. If brand-required, write your own SVG spinner; otherwise accept the Lucide `Loader2Icon` look.
