# Button

A clickable element that triggers an action. Use for form submissions, dialog confirmations, navigation that performs work, and any primary call-to-action.

## When to use

- **Primary actions** — submit a form, confirm a dialog, save changes. Use `variant="default"`.
- **Secondary actions** — cancel, close, dismiss, supporting actions next to a primary button. Use `variant="secondary"` or `variant="outline"`.
- **Destructive actions** — delete, remove, discard, anything that loses data. Use `variant="destructive"`.
- **Tertiary / low-emphasis actions** — toolbar controls, repeated actions in a list. Use `variant="ghost"`.
- **Icon-only triggers** — close buttons, expand/collapse, menu openers. Use `size="icon"` and always set `aria-label`.

## When not to use

- **Inline navigation inside body text** — use the `Link` component, not `variant="link"`.
- **On/off toggles** — use a `Switch` or `Checkbox` instead of a stateful button.
- **Selecting from a list of options** — use `RadioGroup` or `Select`.

## Composition

- Use `asChild` to wrap a router link or anchor without losing styles: `<Button asChild><a href="…">Open</a></Button>`.
- Place icons as children, not via a prop: `<Button><PlusIcon />Create</Button>`. Sizing is handled automatically.
