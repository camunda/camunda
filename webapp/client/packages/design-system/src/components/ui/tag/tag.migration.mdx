# Tag — Carbon → shadcn migration guide

## Naming

- Carbon: `Tag`.
- shadcn: `Badge`.

Same shape (small pill-shaped label), different name.

## Mapping

| Concept | Carbon `Tag` | shadcn `Badge` |
|---|---|---|
| Wrapper | `<Tag type="blue">{text}</Tag>` | `<Badge variant="default">{text}</Badge>` |
| Text content | children | children |
| Colour family | `type="red"/"magenta"/"purple"/"blue"/"cyan"/"teal"/"green"/"gray"/"cool-gray"/"warm-gray"/"high-contrast"/"outline"` (12 variants) | `variant="default"/"secondary"/"destructive"/"outline"/"ghost"/"link"` (6 variants) |
| Filter / dismissible | `filter` + `onClose` props | not modeled — render an inline X button manually |
| Icon | `renderIcon={Icon}` | render the icon as a child of `<Badge>` |
| Sizes | `size="sm"/"md"/"lg"` | none — restyle padding/text-size via Tailwind |
| As link / clickable | `as="a" href` | `<Badge asChild><a href="…">…</a></Badge>` |
| Disabled | `disabled` | `<Badge className="opacity-50 pointer-events-none">` |

## Carbon features missing in shadcn

- **12 colour variants** — Carbon's `type` prop ships full palette (red, magenta, purple, blue, cyan, teal, green, gray, cool-gray, warm-gray, high-contrast, outline). shadcn's `variant` ships 6 (default / secondary / destructive / outline / ghost / link). To match the Carbon palette, extend `badgeVariants` with custom entries:
  ```ts
  // Add to badgeVariants in badge.shadcn.tsx (or override locally)
  red: 'bg-red-100 text-red-900 dark:bg-red-900/30 dark:text-red-200',
  blue: 'bg-blue-100 text-blue-900 dark:bg-blue-900/30 dark:text-blue-200',
  green: 'bg-green-100 text-green-900 dark:bg-green-900/30 dark:text-green-200',
  // etc.
  ```
- **`filter` boolean + built-in close button** — Carbon's filter-tag variant. shadcn requires manual composition:
  ```tsx
  <Badge variant="secondary" className="gap-1 pr-1">
    {label}
    <button aria-label="Remove" onClick={onClose}>
      <X className="size-3" />
    </button>
  </Badge>
  ```
- **`onClose` callback** — paired with `filter`. Use the manual recipe above and own the state.
- **`renderIcon` prop** — Carbon's render-prop for a leading icon. shadcn: just put the icon as a child (`[&>svg]:size-3` is already wired for SVG sizing).
- **`size="sm"/"md"/"lg"`** — Carbon's three density variants. shadcn ships one size; restyle via Tailwind (`text-[10px] px-1.5 py-0` for sm, `text-sm px-3 py-1` for lg).
- **`title` prop** — Carbon's tooltip on hover (showing full text when truncated). shadcn: wrap with `<Tooltip>` if needed.
- **Skeleton variant (`<TagSkeleton>`)** — Carbon ships skeletons. shadcn has none — use the generic `skeleton` primitive.
- **`disabled` prop** — Carbon greys it out. shadcn: `className="opacity-50 pointer-events-none"` or `aria-disabled`.

## shadcn features missing in Carbon

- **`asChild` slot** — `<Badge asChild><a href="…">{label}</a></Badge>` makes the badge a real link without nested elements. Carbon's `as` prop is similar but less idiomatic.
- **`link` variant** — shadcn ships a "link"-styled badge variant (text-coloured, underline-on-hover). Carbon has no equivalent.
- **`ghost` variant** — transparent-bg, hover-only fill. Carbon: would build manually.
- **Tailwind composability** — every prop accepts `className` for one-off tweaks.
- **`focus-visible:ring`** — shadcn adds a focus ring when used as a focusable element (`asChild` link). Carbon's `Tag` uses default focus styles.

## Behavioural differences

- **Default element** — Carbon: `<span>` (or `<button>` if `clickable`). shadcn: `<span>` (or whatever via `asChild`).
- **Default radius** — Carbon: 12px (pill shape, full-rounded). shadcn: `rounded-full` — match.
- **Default padding** — Carbon: ~4px vertical, 8px horizontal. shadcn: `px-2 py-0.5` (~2px vertical, 8px horizontal). Slightly tighter.
- **Default font weight** — Carbon: 400. shadcn: `font-medium` (500).
- **Hover** — Carbon: subtle hover state if interactive. shadcn: `hover:bg-primary/90` etc. when used inside an anchor (via `[a&]:` selectors).

## Migration checklist

1. Rename: `<Tag>` → `<Badge>`.
2. Map `type` → `variant` (best-effort):
   - `gray` / `cool-gray` / `warm-gray` → `secondary`
   - `red` → `destructive`
   - `outline` → `outline`
   - `red`/`magenta`/`purple`/`blue`/`cyan`/`teal`/`green` (semantic colours) → extend `badgeVariants` with project-local custom variants, or apply Tailwind classes inline (`<Badge className="bg-blue-100 text-blue-900">`)
   - `high-contrast` → `default`
3. For `filter` + `onClose`: render an inline X button manually (see canonical recipe above).
4. For `renderIcon={Icon}`: put `<Icon />` as a child of `<Badge>`. Default sizing (`[&>svg]:size-3`) is already applied.
5. For `size`: drop the prop, apply `className` for sm/lg variants.
6. For `title` (tooltip on truncation): wrap with `<Tooltip>` (see `tooltip/migration_guide.md`).
7. For `disabled`: `className="opacity-50 pointer-events-none"` or `aria-disabled`.
8. For Carbon's `as="a" href={url}`: use `<Badge asChild><a href={url}>{label}</a></Badge>`.
9. **If your app uses many semantic colour tags** (e.g., status badges in tables): centralise the colour mapping by extending `badgeVariants` once in `badge.shadcn.tsx` rather than repeating Tailwind classes per call-site.
